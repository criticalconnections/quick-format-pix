# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

Bun is the package manager (`bun.lockb`, `bunfig.toml` with `saveTextLockfile = false`).

- `bun install` — install deps
- `bun run dev` — Vite dev server (also emulates Workers runtime via `@cloudflare/vite-plugin`)
- `bun run build` — production build (Cloudflare Workers bundle)
- `bun run preview` — preview the built bundle
- `bun run lint` — ESLint over the repo
- `bun run format` — Prettier write
- `./install.sh [docker|self-hosted]` — one-shot installer; `./install.sh stop` removes the docker container

There is no test runner configured.

## Architecture

This is **TOOLSHED**, a collection of browser-only utilities (HEIC convert, color extract, password gen, image compress, QR, EXIF strip, base64, favicon). The defining constraint: **everything runs in the user's browser; no uploads, no backend logic**. The "server" is a Cloudflare Worker that only ships the SSR shell.

Stack:
- **TanStack Start** (`@tanstack/react-start`) with file-based routing in `src/routes/` — `routeTree.gen.ts` is generated by `@tanstack/router-plugin`, do not edit by hand.
- **React 19**, **Vite 7**, **Tailwind v4** (no `tailwind.config`; configured via `@tailwindcss/vite` and CSS).
- **`@cloudflare/vite-plugin`** — `wrangler.jsonc` points `main` at `@tanstack/react-start/server-entry`. Build output is a Workers bundle.
- **`@lovable.dev/vite-tanstack-config`** — wraps `defineConfig` and pre-installs tanstackStart, viteReact, tailwindcss, tsConfigPaths, the cloudflare plugin (build-only), componentTagger (dev-only), `VITE_*` env injection, the `@/*` path alias, React/TanStack dedupe, and sandbox port/host detection. **Do not re-add those plugins manually in `vite.config.ts`** — duplicate plugins break the app. Pass extras via `defineConfig({ vite: { ... } })`.
- **shadcn/ui** (`new-york` style, `components.json`) lives under `src/components/ui/`; `@/*` resolves to `src/*`.

Image conversion pipeline (`src/lib/imageConvert.ts`) — see `.lovable/plan.md` for the design rationale. The HEIC decode path tries `heic-to` (libheif 1.21) first, falls back to `heic2any` (libheif 1.10), and surfaces a combined `decoderError` on failure for the Debug panel. Both packages are loaded via dynamic `import(...)` to keep them out of the SSR bundle.

## Conventions

- Tools are top-level routes (`src/routes/<tool>.tsx`); the landing list lives in `src/routes/index.tsx`'s `TOOLS` array — when adding a tool, register it both as a route file and in that array.
- Visual identity is "neo-brutalist": class hooks like `brutal-card`, lime accent (`var(--accent-lime)`), mono + display fonts. Match existing styling rather than introducing a new system.
- Konami sequence and triple-tap on the hero "BIG ATTITUDE." trigger a `BrokenScreen` easter egg — preserve `useKonamiBreak` and the tap state when refactoring `index.tsx`.
- ESLint disables `@typescript-eslint/no-unused-vars` and TS turns off `noUnusedLocals`/`noUnusedParameters`; don't reintroduce them without discussion.

## Deployment

Production target is Cloudflare Workers (`wrangler.jsonc`, `compatibility_flags: ["nodejs_compat"]`). For local/self-hosted use, `install.sh` runs the Vite dev server (which already emulates the Workers runtime) inside Docker or directly on the host.
